NON-BLOCKING (PRODUCTION) 
Wrap send_trace() in a daemon thread to avoid blocking your 
agent: 

import threading 

def send_trace_async(user_input, output, 
                     model, latency_ms): 
    t = threading.Thread( 
        target=send_trace, 
        args=(user_input, output, 
              model, latency_ms), 
        daemon=True) 
    t.start() 

# Returns immediately 
send_trace_async("Balance?", 
    "$5,432.", "gpt-4o", 320)

4.  ENABLE AGENT TRAJECTORY  (OPTIONAL, ~10 MIN) 

WHAT IS TRAJECTORY? 
Trajectory is PRISMtrace's step-by-step execution view. 
When your agent queries a database, checks guardrails, 
and calls an LLM — trajectory shows each step in sequence 
with its latency and output. 

Trajectory is NOT automatic. 
Without the code below, the trajectory tab shows an empty 
state. This section shows exactly how to enable it.

LEVEL 2 — MULTI-STEP (FULL EXAMPLE) 
Each step gets its own trace. All share the same session_id. 

def handle_message(user_input): 
  turn_id = str(uuid.uuid4()) 

  # Step 1 — database query 
  t0 = time.time() 
  data = your_db.query(user_input) 
  ms0 = int((time.time()-t0)*1000) 

  requests.post(TRACE_URL, json={ 
    ..., 
    'model':       'databricks-query', 
    'output_message': f'{len(data)} rows', 
    'latency_ms':  ms0, 
    'metadata': { 
      'step':       'data_retrieval', 
      'step_order': 1, 
      'session_id': turn_id 
    }}) 

  # Step 2 — LLM call 
  t1 = time.time() 
  resp = your_llm.call(data, user_input) 
  ms1 = int((time.time()-t1)*1000) 

  requests.post(TRACE_URL, json={ 
    ..., 
    'model':          'claude-sonnet-4-...', 
    'output_message': resp, 
    'latency_ms':     ms1, 
    'metadata': { 
      'step':       'llm_call', 
      'step_order': 2, 
      'session_id': turn_id  # same! 
    }}) 
  return resp

WHAT EXPORT DESTINATIONS DO 
Export Destinations send your PRISMtrace trace data to 
your data warehouse so your data team can run SQL on it 
and build dashboards in their existing tools. 

ℹ  Not for observing your database 
These send data OUT of PRISMtrace. Observing database 
calls inside your agent uses trajectory instrumentation (Step 
5 above).

workspace 
•  Snowflake  Writes to PRISMTRACE.PUBLIC.TRACES 
via MERGE 
•  Amazon S3  Coming soon — archival and pipeline

AGENT FLEET 
Every agent auto-detected from traces — no manual 
registration. Quality score, guardrail rate, latency, and last 
active shown per agent. Add description, owner, and 
environment via the pencil icon on any card. 

EXAMPLE QUERY AFTER DATABRICKS EXPORT 

SELECT agent_name, 
  COUNT(*) AS traces, 
  ROUND(AVG(quality_score),1) 
    AS avg_quality, 
  SUM(CASE WHEN guardrail_blocked 
    THEN 1 ELSE 0 END) AS blocks 
FROM prismtrace_traces 
WHERE created_at >= 
  DATE_SUB(NOW(), INTERVAL 7 DAY) 
GROUP BY agent_name 
ORDER BY blocks DESC;
