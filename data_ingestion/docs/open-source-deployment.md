# Distributing OpenLogic Finance Locally (A2A Architecture)

OpenLogic Finance is designed as a distributed, modular Agent-to-Agent (A2A) framework. 

To ensure this project remains completely open-source and vendor-agnostic, we intentionally avoid hard dependencies on managed cloud services like Google Cloud Storage (GCS) or Amazon S3 for inter-agent communication. Instead, we use a single `docker-compose.yml` stack to orchestrate everything on your local machine with full data sharing capabilities seamlessly out-of-the-box.

## The Ephemeral Storage Problem
In standard microservice architectures (like Cloud Run), agents run in isolated containers. If the Data Ingestion Agent generates a stock chart `assets/BTC_chart.png` on its own local disk, the downstream Coordinator Agent has no way of reading that image file across the network because it lives inside the other container's hardware.

## The Open-Source Solution: Docker Volumes
Instead of uploading charts to GCS and passing URLs across HTTP, our `docker-compose.yml` mounts a shared physical directory from your laptop directly into the working context `/app/assets` of every agent container simultaneously.

This means the `tools.py` can remain purely vanilla Python. When `data_ingestion_agent` saves a CSV file, Docker intercepts the write and drops the physical file into the `shared-assets` folder on your laptop. When the Coordinator agent needs to read that CSV, it just looks in `/app/assets` and Docker instantly feeds the exact same file back.

## Getting Started

Because all containers expect an API key, you must provide a `.env` file at the root repository level.

**1. Create your root Environment File:**
```bash
echo "GEMINI_API_KEY=your_studio_key" > .env
```

**2. Stand up the Agentic Network:**
```bash
docker compose up --build -d
```
*(The `-d` flag runs the agents in detached mode in the background).*

**3. Verify Output Artifacts:**
Once you trigger an A2A HTTP request to the ingestion agent (e.g., via `adk web` or `curl`), look in your root repository directory. You will see a brand new folder called `shared-assets/`. 

All charts, tables, and CSVs generated by any agent across the network will dynamically appear in that folder!
